## 数据集版本化


### 应用场景


#### 场景1

实验的可重现性，了解哪些具体数据被访问


需求：

可以从元数据中导出被缓存数据的列表


#### 场景2


同一个数据集不同版本之间的差异比较，用来分析模型准确性


#### 场景3

对于大规模小文件场景，如何能够快速准备元数据和数据，避免数据预热时间过长


#### 场景4

数据集存放在不同存储来源（不同地域），如何能够对数据集的来源快速追溯。并且可以在使用数据集的过程中完成数据集的跨地域迁移。


总结下来，一些重要能力和目的：


1. 数据集的管理属性，目标是可复用可追踪：数据集名称，版本，描述， owner，数据来源（存储分散）

2. 数据集的版本化，对于不同版本的数据可以分析（文件数量，最新文件日期，训练时刻访问的文件）

3. 数据集的快速重建：根据备份的元数据，快速重构数据集

4. 数据集的无缝迁移
